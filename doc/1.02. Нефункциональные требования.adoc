=== Нефункциональные требования

Производительность и Масштабируемость::
	* Задержка доставки сообщения не более 200мс
	* Поддержка миллионов сообщений в день (даже если в компании 1000 чел, сообщений может быть достаточно много)
Надежность и Отказоустойчивость::
	* SLA: 99.9% целевой uptime
	* Персистентность: Сообщения не теряются!
	* Аварийное восстановление: Репликация баз данных в разных ЦОДах/зонах доступности
Безопасность::
	* Шифрование (TLS 1.3)
	* Аунтификация SSO
Совместимость::
	* Кроссплатформенность: Клиенты для разных платформ (Mobile, Descktop os, Webclient)
	* API совместимость: RESTful API для интеграций
