=== Риски

==== Инфраструктурные риски

[cols="3,^2,^8"]
|===
|Риск |Влияние | Описание

|Зависимость от Keycloak/LDAP (SPOF для аутентификации)
|Критическое
|Keycloak является единственным источником аутентификации и авторизации. При недоступности Keycloak или LDAP/AD все пользователи не смогут войти в систему. API Gateway и WebSocket Gateway не смогут валидировать JWT токены, что приведет к полной остановке сервиса.

|Деградация при потере Kafka
|Высокое
|Kafka используется как шина событий для доставки сообщений в реальном времени (WebSocket Gateway), индексации (Search Service) и аудита. При недоступности Kafka:
- Пользователи не будут получать сообщения в реальном времени
- Поиск по сообщениям перестанет обновляться
- Аудит событий будет потерян
- Chat Service может перестать обрабатывать новые сообщения

|Reconnect storm WebSocket при сетевых проблемах
|Высокое
|При сетевых сбоях или перезагрузке WebSocket Gateway тысячи клиентов могут одновременно попытаться переподключиться, создавая "reconnect storm". Это приведет к:
- Перегрузке WebSocket Gateway
- Превышению rate limits
- Деградации KeyDB (хранение сессий)
- Увеличенной нагрузке на Keycloak (валидация JWT)

|Рост объёма S3 при недостаточном lifecycle management
|Среднее
|S3 хранит все файловые вложения. Без корректной политики lifecycle:
- Неконтролируемый рост затрат на хранение
- Увеличение времени бэкапов и восстановления
- Сложности с соблюдением retention policies
- Потенциальные проблемы с комплаенс

|Недоступность KeyDB
|Среднее
|KeyDB используется для хранения presence, WebSocket сессий и кэша профилей. При отказе:
- Потеря информации о статусе online/offline пользователей
- Дублирование push-уведомлений
- Невозможность корректной работы WebSocket соединений

|Отказ PostgreSQL primary
|Критическое
|PostgreSQL является источником истины для всех бизнес-данных. При отказе primary без успешного failover:
- Невозможность создания/редактирования сообщений
- Потеря доступа к истории чатов
- Остановка всех CRUD операций
- Потенциальная потеря данных при отсутствии репликации

|Сетевая связанность между микросервисами
|Высокое
|При потере связности между сервисами:
- Chat Service не сможет публиковать события в Kafka
- WebSocket Gateway не будет доставлять сообщения
- Search Service не будет индексировать новые сообщения
- File Service не сможет загружать файлы
|===

==== Операционные риски

[cols="3,^2,^8"]
|===
|Риск |Влияние | Описание

|Злоупотребление загрузкой файлов
|Среднее
|Отсутствие строгих лимитов на размер и тип файлов может привести к:
- Быстрому заполнению S3 хранилища
- Увеличению затрат
- Потенциальным атакам через вредоносные файлы
- Деградации File Service

|Неконтролируемый рост количества WebSocket соединений
|Среднее
|При отсутствии корректного управления сессиями:
- Утечка памяти в WebSocket Gateway
- Превышение лимитов KeyDB
- Деградация производительности
- Проблемы с масштабированием

|Отказ сторонних push-сервисов (APN/FCM)
|Низкое
|При недоступности APN или FCM:
- Пользователи не будут получать push-уведомления
- Возможна потеря важных сообщений для offline пользователей
- Необходимость альтернативных каналов уведомлений

|Инциденты безопасности в Keycloak/LDAP
|Критическое
|Компрометация Keycloak или LDAP может привести к:
- Несанкционированному доступу ко всем данным
- Краже учетных данных пользователей
- Модификации прав доступа
- Потере доверия к системе

|Ошибки в конфигурации Kafka retention
|Среднее
|Некорректная настройка retention может привести к:
- Потере событий для replay
- Чрезмерному использованию дискового пространства
|===
