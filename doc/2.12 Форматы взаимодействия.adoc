=== Форматы взаимодействия

Раздел фиксирует контракты обмена данными между клиентами и backend сервисами TeamBridge.

==== Общие правила контрактов

* Транспорт для внешних API - HTTPS + JSON (`UTF-8`)
* Транспорт для realtime - WebSocket Secure (`WSS`) + JSON events
* Транспорт для межсервисных событий - Kafka (`JSON`)
* Транспорт для межсервисных синхронных взаимодействий - HTTP + JSON
* Идентификаторы - `UUID v7`

==== REST API - пример отправки сообщения

Endpoint::
* `POST /api/v1/chats/{chat_id}/messages`

Headers::
* `Authorization: Bearer <JWT>`
* `Content-Type: application/json`
* `X-Request-Id: <uuid>`

Request body::

[source,json]
----
{
  "client_message_id": "f9f8cf2c-473f-4177-a5dd-1e30d3bb9f6e",
  "text": "Привет! Документы отправил.",
  "attachments": [
    {
      "file_id": "5e17f0e3-3d07-426e-b45f-4939429e045a"
    }
  ]
}
----

Response body (`200 OK`)::

[source,json]
----
{
  "message_id": "45883502-d9ca-43be-bf42-57ef7f0e3f6b",
  "chat_id": "f0952258-95c8-4db3-83ca-66f1a246b9be",
  "sender_id": "8dcb22d0-f626-4135-a411-341dc9cb5f74",
  "text": "Привет! Документы отправил.",
  "attachments": [
    {
      "file_id": "5e17f0e3-3d07-426e-b45f-4939429e045a"
    }
  ],
  "created_at": "2026-03-27T10:15:30Z",
  "status": "accepted"
}
----

==== gRPC контракт между API Gateway и Chat Service (.proto пример)

Пример синхронного межсервисного взаимодействия для операций отправки и чтения сообщений::

[source,proto]
----
syntax = "proto3";

package teambridge.chat.v1;

option go_package = "github.com/teambridge/api/gen/chat/v1;chatv1";

service ChatService {
  rpc SendMessage(SendMessageRequest) returns (SendMessageResponse);
  rpc GetMessages(GetMessagesRequest) returns (GetMessagesResponse);
  rpc MarkAsRead(MarkAsReadRequest) returns (MarkAsReadResponse);
}

message SendMessageRequest {
  string request_id = 1;          // трассировка и идемпотентность gateway уровня
  string chat_id = 2;             // UUID v7
  string sender_id = 3;           // UUID v7
  string client_message_id = 4;   // UUID v7 - идемпотентность клиентской команды
  string text = 5;
  repeated Attachment attachments = 6;
  repeated string mentions = 7;   // user_id[]
}

message SendMessageResponse {
  string message_id = 1;          // UUID v7
  string chat_id = 2;
  string sender_id = 3;
  string text = 4;
  repeated Attachment attachments = 5;
  string created_at = 6;          // ISO-8601 UTC
  MessageStatus status = 7;
}

message GetMessagesRequest {
  string request_id = 1;
  string chat_id = 2;
  int32 limit = 3;                // max 100
  string before_message_id = 4;   // cursor pagination
}

message GetMessagesResponse {
  repeated Message items = 1;
  string next_cursor = 2;
}

message MarkAsReadRequest {
  string request_id = 1;
  string chat_id = 2;
  string user_id = 3;
  string last_read_message_id = 4;
}

message MarkAsReadResponse {
  bool success = 1;
  string read_at = 2;             // ISO-8601 UTC
}

message Message {
  string message_id = 1;
  string chat_id = 2;
  string sender_id = 3;
  string text = 4;
  repeated Attachment attachments = 5;
  string created_at = 6;
  string edited_at = 7;
  bool is_deleted = 8;
}

message Attachment {
  string file_id = 1;
}

enum MessageStatus {
  MESSAGE_STATUS_UNSPECIFIED = 0;
  MESSAGE_STATUS_ACCEPTED = 1;
  MESSAGE_STATUS_REJECTED = 2;
}
----

==== WebSocket формат событий

Envelope для всех WS сообщений::

[source,json]
----
{
  "event_type": "message.new",
  "schema_version": 1,
  "event_id": "deca1ea0-61f8-41cb-bfbc-91f4f1cf9168",
  "occurred_at": "2026-03-27T10:15:30Z",
  "payload": {}
}
----

Событие доставки нового сообщения (`message.new`)::

[source,json]
----
{
  "event_type": "message.new",
  "schema_version": 1,
  "event_id": "deca1ea0-61f8-41cb-bfbc-91f4f1cf9168",
  "occurred_at": "2026-03-27T10:15:30Z",
  "payload": {
    "message_id": "45883502-d9ca-43be-bf42-57ef7f0e3f6b",
    "chat_id": "f0952258-95c8-4db3-83ca-66f1a246b9be",
    "sender_id": "8dcb22d0-f626-4135-a411-341dc9cb5f74",
    "text": "Привет! Документы отправил.",
    "created_at": "2026-03-27T10:15:30Z"
  }
}
----

Ack от клиента::

[source,json]
----
{
  "event_type": "message.ack",
  "schema_version": 1,
  "event_id": "9ef5f1f5-bb5e-4281-973d-bfd7cefc8e4e",
  "occurred_at": "2026-03-27T10:15:31Z",
  "payload": {
    "message_id": "45883502-d9ca-43be-bf42-57ef7f0e3f6b",
    "received": true
  }
}
----

==== Kafka формат доменных событий

Topic naming::
* `teambridge.chat.message.v1`
* `teambridge.chat.presence.v1`

Ключ сообщения Kafka::
* Для message событий используется `chat_id` - обеспечивает порядок внутри чата

Событие `MessageCreated`::

[source,json]
----
{
  "event_type": "MessageCreated",
  "schema_version": 1,
  "event_id": "a3fc8a72-1733-4067-b54e-386a89712ed2",
  "occurred_at": "2026-03-27T10:15:30Z",
  "chat_id": "f0952258-95c8-4db3-83ca-66f1a246b9be",
  "message_id": "45883502-d9ca-43be-bf42-57ef7f0e3f6b",
  "sender_id": "8dcb22d0-f626-4135-a411-341dc9cb5f74",
  "text": "Привет! Документы отправил.",
  "attachments": [
    {
      "file_id": "5e17f0e3-3d07-426e-b45f-4939429e045a"
    }
  ],

}
----

==== Версионирование и совместимость

* Backward compatibility - обязательна для минорных изменений
* Новые поля добавляются как optional
* Удаление или изменение семантики поля - только через новую мажорную версию (`v2`)

==== Безопасность и валидация

* JWT обязателен для всех внешних REST/WS вызовов
* Проверка прав доступа выполняется на стороне самих сервисов
* Максимальный размер payload для текстового сообщения - 16 KB
* Для вложений передается только `file_id`, бинарные данные хранятся в объектном хранилище
