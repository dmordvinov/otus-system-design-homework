=== Планы тестирования

Цель тестирования - подтвердить, что TeamBridge выполняет функциональные и нефункциональные требования по realtime обмену сообщениями, надежности, безопасности и масштабируемости.

==== Объем и границы тестирования

В объеме текущего этапа::
* API Gateway, WebSocket Gateway, Chat Service, Search Service, File Service, Notification Service
* Интеграции с Keycloak/LDAP, Kafka, PostgreSQL, KeyDB/Redis, S3/Object Storage
* Web/Desktop/Mobile клиенты в части backend контрактов

Вне объема текущего этапа::
* Глубокое UX тестирование клиентских приложений
* Локализация и A/B тесты интерфейса

==== Уровни тестирования

1. Unit тесты::
* Проверка бизнес-правил и доменной логики сервисов
* Валидация контрактов DTO, схем и преобразований
* Проверка идемпотентности обработки `client_message_id`

2. Integration тесты::
* Контракты между сервисами (REST/gRPC/WebSocket/Kafka)
* Проверка работы с PostgreSQL, KeyDB/Redis, S3
* Проверка событийного потока `MessageCreated -> consumers`

3. End-to-End тесты::
* Пользовательские сценарии от логина до доставки сообщения
* Сценарии online доставки через WSS и offline fallback через push
* Сценарии с файлами: upload -> message reference -> download

4. Non-functional тесты::
* Нагрузочные и стресс тесты
* Тесты отказоустойчивости и восстановления
* Тесты безопасности
