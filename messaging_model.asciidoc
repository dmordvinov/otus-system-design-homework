== Связи между сущностями

[source,plantuml]
----
@startuml
UserProfile --> Message : отправляет
Message --> Chat : принадлежит
Chat --> ChatMember : участники
ChatMember --> UserProfile : пользователь
Message --> Attachment : содержит
Attachment --> File : ссылается на
File --> FileMetadata : метаданные
Chat --> Message : последние сообщения
@enduml
----

== Описание сущностей

=== UserProfile
- Хранит информацию о пользователе, импортированного из Keycloak
- Не редактируется, кроме статуса присутствия
- Содержит основную информацию: имя, email, аватар

=== Message
- Содержит текстовое сообщение с различными типами контента
- Поддерживает редактирование сообщений
- Может содержать вложения и реакции
- Типы сообщений: текст, эмодзи, изображения, файлы

=== Chat
- Поддерживает личные и групповые чаты
- Групповые чаты имеют администраторов
- Хранит историю переписки

=== Attachment
- Прикрепленные файлы к сообщениям
- Ссылается на внешнюю систему хранения файлов

=== File
- Файлы, хранимые в отдельной системе
- Содержит метаданные для разных типов файлов
- Поддерживает различные форматы (изображения, документы и т.д.)

== Технические детали

=== UUID
Все сущности используют UUID для уникальной идентификации

=== Временные метки
- createdAt: время создания сущности
- updatedAt: время последнего обновления
- timestamp: время отправки сообщения
- editedAt: время редактирования сообщения

=== Статусы
- PresenceStatus: статус присутствия пользователя
- MemberRole: роль пользователя в чате
- MessageType: тип содержимого сообщения

== Пример использования

[source,plantuml]
----
@startuml
actor User
participant "Messaging Service" as Service
participant "Database" as DB
participant "File Storage" as FS

User -> Service : Отправить сообщение
Service -> DB : Сохранить сообщение
Service -> FS : Загрузить файл (если есть)
Service -> DB : Сохранить вложение
Service -> DB : Обновить последнее сообщение в чате
@enduml
----

== Дополнительные возможности

=== Реакции на сообщения
- Пользователи могут добавлять реакции (эмодзи) к сообщениям
- Реакции хранятся отдельно от основного содержимого

=== Редактирование сообщений
- Сообщения могут быть отредактированы до определенного времени
- История редактирования сохраняется

=== Групповые чаты
- Администраторы могут управлять участниками
- Возможность назначения администраторов
- Права участников различаются по роли

== Бизнес-логика

=== Состояния чатов
- DIRECT: личные чаты между двумя пользователями
- GROUP: групповые чаты с несколькими участниками

=== Права пользователей
- MEMBER: обычный участник
- ADMIN: может управлять участниками и настройками
- OWNER: полные права на чат

=== Типы сообщений
- TEXT: текстовые сообщения
- EMOJI: реакции с эмодзи
- IMAGE: изображения
- FILE: прикрепленные файлы
- SYSTEM: системные сообщения
